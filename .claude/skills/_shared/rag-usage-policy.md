# RAGデータ運用ポリシー（koyama-x-post / koyama-yt-script 共通）

このリポジトリ単体のルールです。**他リポジトリとの同期や共通化は想定しません。**

方針：**`insights/_routing.md` で仕訳 → 優先ファイルを読む → 足りなければ grep**  
使用履歴ログ・`_used`・クールダウン・ファイル移動は行わない。

---

## コーパス

| 場所 | 役割 |
|------|------|
| `insights/_routing.md` | **必ず最初に読む** RAG 仕訳インデックス |
| `insights/*_analysis.txt` 等 | 補助コーパス（外注・AI・共通事例） |
| `insights/video_prompt/` | YouTube 台本プロンプトの配置先 |
| `小山さん/*.txt` | 主コーパス（小山さん固有の言葉・型・マーケティング事例） |

---

## 4原則

1. RAG は **広く取得してよい**。**grep より先に `_routing.md` を読む**。
2. 記事・台本に**明示するコア事例は少数**（構成の質のため）。
3. **テーマ一貫性**を最優先にし、事例の羅列をしない。
4. **ファイルのコピー・退避・履歴ファイルの更新はしない**。Read / Grep の参照のみ。

---

## 取得と反映

### 取得フェーズ

- Read: `insights/_routing.md`
- Read: 主カテゴリの優先ファイル 5〜8 件（`_routing.md` に書かれた `小山さん/` または `insights/` のパス）
- 不足時: Grep で `小山さん/*.txt` と `insights/*.txt`

### 選定・反映フェーズ

**koyama-x-post（X記事）**

- コア具体事例 **1〜2件**。その他は口調・論理の参照にとどめる。

**koyama-yt-script（YouTube台本）**

- 本編コア事例 **2〜3件**。Shorts3本は **本編で採用した事例を別角度で使い回す**（新規事例を増やさない）。手順の詳細は `koyama-yt-script/SKILL.md`。
